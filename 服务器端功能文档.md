# Keylogger 服务器端功能需求文档

## 1. 概述

本文档详细描述了 Keylogger 服务器端网页需要实现的功能，以便管理和控制多个客户端 Keylogger 程序。

## 2. 核心功能模块

### 2.1 客户端发现与管理

#### 2.1.1 客户端发现
- **功能描述**：自动或手动发现网络中的 Keylogger 客户端
- **实现方式**：
  - 支持手动输入 IP 地址和端口连接
  - 支持局域网扫描（可扫描常见端口范围）
  - 客户端通过 `ping` 命令响应发现请求

#### 2.1.2 客户端列表
- **功能描述**：显示所有已发现和连接的客户端
- **显示信息**：
  - 客户端 IP 地址
  - 本地监听端口
  - 客户端状态（在线/离线）
  - 录制状态（录制中/已暂停）
  - 上传状态（启用/禁用）
  - 最后连接时间
  - 日志文件目录

#### 2.1.3 客户端详情
- **功能描述**：查看单个客户端的详细信息
- **详细信息**：
  - 基本信息（IP、端口、状态等）
  - 系统信息（可选）
  - 历史连接记录
  - 已上传的日志文件列表

### 2.2 客户端控制功能

#### 2.2.1 录制控制
- **暂停录制**：发送 `{"action":"pause_record"}` 命令
- **恢复录制**：发送 `{"action":"resume_record"}` 命令
- **实时反馈**：显示当前录制状态

#### 2.2.2 上传控制
- **开始定时上传**：发送 `{"action":"start_upload"}` 命令
- **停止上传**：发送 `{"action":"stop_upload"}` 命令
- **立即上传**：发送 `{"action":"upload_once"}` 命令
- **上传进度显示**：显示上传状态和结果

#### 2.2.3 服务器配置
- **功能描述**：修改客户端的上传服务器地址
- **命令格式**：`{"action":"set_server", "host":"x.x.x.x", "port":5244}`
- **验证**：发送后通过 `get_status` 验证配置是否生效

### 2.3 日志文件管理

#### 2.3.1 日志文件列表
- **功能描述**：显示客户端已上传的日志文件
- **文件信息**：
  - 文件名（格式：IP_YYYYMMDD.log）
  - 文件大小
  - 上传时间
  - 文件预览链接

#### 2.3.2 日志文件查看
- **功能描述**：在线查看日志文件内容
- **支持功能**：
  - 文本格式显示
  - 关键词搜索
  - 按时间筛选
  - 下载原始文件

#### 2.3.3 日志文件下载
- **功能描述**：下载单个或批量日志文件
- **支持格式**：原始 .log 文件
- **批量下载**：支持选择多个文件打包下载

### 2.4 通信协议实现

#### 2.4.1 TCP 连接管理
- **功能描述**：与客户端建立和维护 TCP 连接
- **实现要点**：
  - 支持多客户端同时连接
  - 连接超时处理
  - 自动重连机制
  - 连接状态监控

#### 2.4.2 命令发送与响应
- **功能描述**：发送 JSON 命令并处理响应
- **命令格式**：JSON 格式，以换行符 `\n` 结束
- **响应处理**：
  - 解析 JSON 响应
  - 错误处理
  - 超时处理

#### 2.4.3 心跳检测
- **功能描述**：定期检测客户端是否在线
- **实现方式**：
  - 定期发送 `{"action":"ping"}` 命令
  - 根据响应更新客户端状态
  - 超时判定为离线

## 3. 通信协议详解

### 3.1 命令格式

所有命令均为 JSON 格式，以换行符 `\n` 结束。

### 3.2 可用命令列表

| 命令 | 说明 | 响应类型 |
|------|------|----------|
| `{"action":"ping"}` | 心跳检测，获取客户端状态 | `pong` |
| `{"action":"start_upload"}` | 开始定时上传日志 | `upload_started` |
| `{"action":"stop_upload"}` | 停止上传 | `upload_stopped` |
| `{"action":"upload_once"}` | 立即上传一次 | `upload_completed` |
| `{"action":"pause_record"}` | 暂停录制 | `record_paused` |
| `{"action":"resume_record"}` | 恢复录制 | `record_resumed` |
| `{"action":"get_status"}` | 获取完整状态信息 | `status` |
| `{"action":"set_server", "host":"x.x.x.x", "port":5244}` | 设置上传服务器地址 | `server_configured` |

### 3.3 响应格式

成功响应示例：
```json
{
  "status": "ok",
  "type": "pong",
  "data": {
    "recording": true,
    "upload_enabled": false,
    "local_port": 54321,
    "ip": "10.88.1.100"
  }
}
```

错误响应示例：
```json
{
  "status": "error",
  "message": "unknown action"
}
```

## 4. 数据库设计（可选）

### 4.1 客户端表
```
clients (
  id INT PRIMARY KEY AUTO_INCREMENT,
  ip VARCHAR(45) NOT NULL,
  port INT NOT NULL,
  status ENUM('online', 'offline') DEFAULT 'offline',
  recording BOOLEAN DEFAULT false,
  upload_enabled BOOLEAN DEFAULT false,
  last_seen DATETIME,
  log_dir VARCHAR(255),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### 4.2 日志文件表
```
log_files (
  id INT PRIMARY KEY AUTO_INCREMENT,
  client_id INT,
  filename VARCHAR(255) NOT NULL,
  file_size BIGINT,
  upload_time DATETIME,
  file_path VARCHAR(255),
  FOREIGN KEY (client_id) REFERENCES clients(id)
)
```

## 5. 用户界面建议

### 5.1 主界面布局
1. **顶部导航栏**：Logo、用户信息、设置
2. **左侧边栏**：
   - 客户端列表
   - 日志文件管理
   - 系统设置
3. **主内容区**：根据当前选择显示相应内容

### 5.2 客户端列表界面
- 表格形式展示所有客户端
- 支持搜索和筛选
- 每行提供快捷操作按钮（连接、断开、控制）
- 状态指示器（在线/离线用不同颜色表示）

### 5.3 客户端详情界面
- 标签页布局：
  - 概览：基本信息和状态
  - 控制：录制和上传控制按钮
  - 日志：已上传的日志文件列表
  - 历史：连接和操作历史

## 6. 技术栈建议

### 6.1 后端技术
- **Web 框架**：Node.js (Express) / Python (FastAPI/Flask) / Go (Gin)
- **数据库**：MySQL / PostgreSQL / SQLite
- **TCP 通信**：原生 Socket 编程

### 6.2 前端技术
- **框架**：React / Vue.js / Angular
- **UI 组件库**：Ant Design / Element UI / Bootstrap
- **实时通信**：WebSocket（用于实时更新客户端状态）

## 7. 安全考虑

1. **身份认证**：服务器端需要用户登录认证
2. **访问控制**：不同用户有不同权限
3. **数据加密**：日志文件存储加密
4. **通信加密**：建议使用 TLS 加密 TCP 连接
5. **审计日志**：记录所有操作行为

## 8. 扩展功能（可选）

1. **告警通知**：客户端上线/离线、异常情况通知
2. **统计分析**：日志数据分析、可视化
3. **批量操作**：同时控制多个客户端
4. **定时任务**：定时上传、定时扫描
5. **API 接口**：提供 RESTful API 供第三方集成
